# sapho

**A single source of truth for your products.** One markdown truth file per product; every
AI you work with reads it before it writes, and folds new decisions back in.

This repo is the open-source home of the sapho **conventions**: the install spec that AI
agents follow, and the site at [sapho.xyz](https://sapho.xyz) that serves it. It contains
no user data — every user's corpus (their actual truth files) lives in their own private
git repo.

## Install sapho

Give one sentence to your AI (Claude Code, ChatGPT, Cursor, or anything that can read a
page):

> Read https://sapho.xyz/install and set up sapho for me.

Your AI reads the spec, checks for an existing corpus before creating anything, shows you
its plan, and sets everything up. Free. Plain files, on your machine, under your control.

## What's in here

- `index.html` — the marketing page (human face)
- `install.md` — the capability-detecting installation spec (agent face); HTML mirror
  generated by `scripts/gen_install_html.py`
- `llms.txt` — agent discovery pointer
- `robots.txt`, `vercel.json` — hosting plumbing

## How it works, in three lines

- **the files** — one truth file per product, plain markdown, in a git repo you own
- **the skills** — conventions that make every AI read the truth first and write decisions back
- **git** — history, sync, and one canonical copy every surface reads

## Contributing

Per-surface adapters are the most useful contribution: setup instructions for AI tools the
spec doesn't cover yet (new agents, new chat apps, new IDEs). Open a PR against
`install.md` — keep the capability-detection structure and the discover-before-create rule
intact.
